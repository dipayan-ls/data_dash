# Data Integrity Dashboard

A unified data-integrity monitoring tool that fetches advertising data from **9 ad platform APIs**, compares it against **BigQuery warehouse data**, and visualises discrepancies in an interactive React dashboard.

## Running Locally

### 1 — Prerequisites

```bash
# Python deps
pip install -r requirements.txt

# Node / frontend deps
npm install
```

### 2 — Set up credentials

Copy `.env.example` to `.env` and fill in your BigQuery service account:

```bash
cp .env.example .env
```

At minimum you need **one** of:

| Variable | Description |
|---|---|
| `BIGQUERY_SERVICE_ACCOUNT_JSON` | Full JSON string of service account |
| `BIGQUERY_SERVICE_ACCOUNT_FILE` | Path to JSON key file |

Or place the key at `./credential_file.json` (the default path, already in `.gitignore`).

### 3 — Start Flask backend (Terminal 1)

```bash
python3 app.py
# → Running on http://0.0.0.0:3000
```

### 4 — Start Vite frontend (Terminal 2)

```bash
npm run dev
# → Local: http://localhost:5173
```

Open http://localhost:5173 in your browser.

> **Note:** Vite proxies all `/api` requests to the Flask backend on `:3000` automatically.

## Using the Dashboard

### Option A — Run the Auto Scraper (requires backend + credentials)

1. Click **"Run Auto Scraper"** in the sidebar.
2. Select a date range, workspaces, and channels.
3. Click **"Run Scraper"** → data loads automatically into the dashboard.

### Option B — Upload an Existing CSV

1. Click **"Choose File"** in the sidebar.
2. Upload any `combined_metrics_*.csv` generated by `combined_scraper.py`.

### Option C — CLI (no frontend needed)

```bash
python3 combined_scraper.py
# Interactive prompts will guide you through workspace/channel/date selection
```

## Configuration

### BigQuery Credentials (priority order)

1. `BIGQUERY_SERVICE_ACCOUNT_JSON` env var (full JSON string)
2. `BIGQUERY_SERVICE_ACCOUNT_FILE` env var (path to file)
3. `./credential_file.json` (default, relative to project root)

### Channel Credential Overrides (optional)

See `.env.example` for the full list. All channel credentials are loaded from **Google Cloud Datastore** by default.

## Credential Extraction Utility

`scraper/ndb_n.py` can dump all platform credentials for a workspace from Datastore to a CSV:

```bash
cd data-integrity-dashboard
python3 scraper/ndb_n.py
# Enter workspace namespace: ls_turn5_d73563ce
# → Saves channel_credentials.csv to project root
```

## Error Handling

- **403 / 401 / 429** (Google Ads): Account is skipped with a warning; scraper continues.
- All other channels: errors are caught and printed; scraper continues.
- **Amazon**: max 90-day date range per report — longer ranges are auto-split into chunks.
- **Reddit**: fetched day-by-day for completeness.
- **Twitter/X**: API returns `TOTAL` granularity — totals are distributed evenly per day.
